### 1.1.1 Web简介 3
Web (World Wide Web, WWW) 全球广域网或万维网，俗称网站（WebSite）。

#### 1. 网站

HTPP 的9种请求方式
请求方式 | 功能说明
---|---
GET     | 请求指定资源地址的网页信息，并返回实体数据
HEAD    | 与GET请求类似，返回的响应中没有具体内容，用于获取响应头数据
POST    | 向指定资源地址提交数据进行处理请求（如表单、文件）
PUT     | 向指定资源地址上传数据内容
DELETE  | 向指定资源地址发送删除资源的请求
CONNECT | HTTP/1.1 协议中预留给能够将连接改为管道试的代理服务器
OPTIONS | 获取服务器针对特定资源所支持的HTML请求方法
TRACE   | 回复并显示服务器收到的请求，用于测试和诊断
PATCH   | 是对PUT方法的补充，用来对已知资源进行局部更新

#### 4. HTTP状态码
HTTP中常用的状态码
状态码 | 英文名称 | 说明
----|----|----
100 | Continue      | 100及以上状态码用于信息的持续操作响应，需求请求后继续执行相关操作。
200 | OK            | 默认状态码。表示操作成功并返回对应的响应对象数据给客户端。
201 | Created       | 表示服务器已经创建了新的资源。
204 | No Context    | 表示服务器成功处理，但是没有返回内容给客户端。
300 |               | 300及以上状态码用于重定向，具有这些状态码响应可能有或者没有可能没有响应对象的业务数据。
400 | No Found      | 400及以上状态码用于访问服务器端发生异常时，返回给客户端的响应状态码。
404 |               | 客户访问服务端时，无法找到对应的资源（主要是网页）
500 |               | 500及以上状态码用于服务端内部错误提示。
